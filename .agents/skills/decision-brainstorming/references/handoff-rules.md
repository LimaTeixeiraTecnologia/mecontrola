# Regras de Handoff

## Para `technical-discovery-production`
Use quando a decisão recomendada envolve arquitetura, viabilidade técnica, operação, segurança, confiabilidade, observabilidade, escalabilidade, volumetria ou custo de produção.

## Para `epic-story-discovery`
Use quando a decisão recomendada já define direção suficiente para refinar uma capacidade de produto, decompor épico e mapear user stories.

## Para `tracker-to-prd`
Use quando existe uma US ou épico no Jira ou Azure DevOps e o objetivo é transformar a decisão preliminar em contexto para PRD preservando origem de tracking.

## Regra Geral
Não executar o handoff automaticamente. Informar o bundle e a próxima skill recomendada.

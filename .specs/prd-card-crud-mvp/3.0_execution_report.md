## Algoritmo InvoiceFor

Implementado conforme ADR-002:
1. Normalização: `purchase.In(tz)` + extração de dia como `time.Date(..., 0, 0, 0, tz)` para comparação correta
2. `computeCycle(cycle, year, month, tz)` calcula closing+due com clamp antes do avanço
3. Auto-detect: closing>due → due no mês seguinte; closing<due → due no mesmo mês; closing==due → closing=due-1 dia
4. Se `purchaseDay.After(closingDate)` → avança um ciclo
5. Retorna `Invoice{ClosingDate, DueDate}` em SP

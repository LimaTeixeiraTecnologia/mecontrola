## Suposições
- `RunHousekeepingForTest` e `RunReaperForTest` são métodos exportados para viabilizar testes unitários sem depender do scheduler real (robfig/cron não expõe API de trigger manual). Padrão documentado no godoc de cada método como "não deve ser usado em produção".
- `ClassifyErrorForTest` exportado pelo mesmo motivo: `classifyError` é privada por OC #9 mas precisa de cobertura tabular.
- `OutboxMetrics` usa `UpDownCounter` para `deliveries.pending` (em vez de `Gauge` com callback) porque o provider devkit-go usa callback sem suporte a labels dinâmicas por subscription. Callers passam delta diretamente.
- `SetPending` aceita delta (não valor absoluto) porque o caller (Dispatcher/Subsystem) mantém o estado anterior.
- `isPanicError` retorna sempre false porque panics são wrappados como `ErrPermanent` pelo Dispatcher — detectados via `errors.Is(err, ErrPermanent)` na linha acima.

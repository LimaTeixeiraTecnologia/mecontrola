## Riscos Residuais
- OutboxMetrics não está conectada ao Dispatcher na versão atual: o Dispatcher usa NoopMetrics() pois OutboxMetrics tem interface diferente. Métricas de delivery (processed/failed/dlq) não são emitidas até que a task 9.0 conecte as interfaces. Impacto: observabilidade parcial em produção até 9.0 estar done.
- A função `max` local em subsystem.go pode entrar em conflito com o builtin `max` do Go 1.21+ em contextos futuros; monitorar ao atualizar versão do compilador.

## Suposições
- `outbox.Subsystem` não implementa a interface `Metrics` diretamente — usa `NoopMetrics()` como placeholder pois `OutboxMetrics` tem assinatura diferente; a instrumentação real de métricas pode ser conectada na task 9.0.
- A função `max(a, b time.Duration)` foi definida localmente em subsystem.go pois Go 1.21+ já tem `max` builtin mas para Duration precisamos de uma específica; o compilador aceita sem shadowing.
- O runtime test para ModeWorker foi atualizado: antes esperava `runOK: true` (sem subsistemas), agora espera `runOK: false` (Dispatcher precisa de DB real).
- `registerSubscriptions` usa `"outbox-dummy"` como nome (sem prefixo `outbox.` para respeitar o regex `^[a-z][a-z0-9_-]{2,63}$`).

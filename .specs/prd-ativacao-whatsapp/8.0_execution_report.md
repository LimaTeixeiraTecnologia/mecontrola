# Execution Report — Task 8.0: E-mail para `/ativar`, estado do token e timestamps server-side

## Status: done

## Subtarefas implementadas

### 8.1 — URL do CTA no e-mail aponta para página de ativação

- `ActivationTemplateInput.WaMeURL` renomeado para `ActivationURL`
- Formato: `${ONBOARDING_ACTIVATION_PAGE_URL}/ativar?token=<clearToken>`
- Ex.: `https://mecontrola.app.br/ativar?token=AAAA...`
- `SupportURL` continua sendo `wa.me/{bot}` sem query param (suporte humano)
- `module.go` passa `cfg.ActivationPageURL` como `activationPageURL` no construtor

### 8.2 — Supressão de e-mail quando assinatura já vinculada (recompra)

- Nova interface `appinterfaces.SubscriptionBoundChecker` em `internal/onboarding/application/interfaces/subscription_bound_checker.go`
- Implementação Postgres em `internal/onboarding/infrastructure/repositories/postgres/subscription_bound_checker.go`
  - Query: `SELECT user_id IS NOT NULL FROM billing_subscriptions WHERE id = $1`
  - `sql.ErrNoRows` → `false, nil` (assinatura inexistente = não vinculada)
- `SendActivationEmail.Execute` checa bound antes de render/send quando `boundChecker != nil` e `SubscriptionID != ""`
- `ActivationEmailConsumer` passa `p.SubscriptionID` no input do use case
- `module.go` injeta `postgres.NewSubscriptionBoundChecker(o11y, db)` no construtor

### 8.3 — `wa_me_url` em `get_token_state`: "Oi" quando há telefone, token cru no edge case

- Função pura `buildWaMeURL(botNumber, clearToken, customerMobileE164 string) string` adicionada
- Telefone presente → `https://wa.me/{bot}?text=Oi`
- Sem telefone (edge case) → `https://wa.me/{bot}?text={clearToken}` (bare token)
- Nenhuma referência a "ATIVAR" no código de produção
- Nenhum `telegram_deep_link` (output DTO não tem o campo)

### 8.4 — Persistência de `email_sent_at` e `activation_started_at` (set-once-if-null)

- `MagicTokenRepository.UpdateSetEmailSentAt(ctx, tokenID, now)` adicionado à interface e implementação Postgres
- SQL idempotente: `COALESCE(email_sent_at, $1)` e `COALESCE(activation_started_at, $1)`
- Chamado após envio bem-sucedido do e-mail; erro registrado no span mas não propagado (não bloqueia o fluxo)
- Mock `mocks/magic_token_repository.go` já possuía o método (adicionado por tarefa prévia)

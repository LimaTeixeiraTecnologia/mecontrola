## Contrato Implementado

- Endpoint: `POST /api/v1/onboarding/tokens/{token}/opened`
- Body: `{"event":"page_opened"|"whatsapp_opened"}`
- Resposta: `204 No Content` sempre (token inválido, evento inválido, erro de DB — todos retornam 204)
- Timestamps `page_opened_at` / `whatsapp_opened_at` são set-once-if-null via `WHERE IS NULL`
- Rate limit via `middleware.NewRateLimiter` com config `ONBOARDING_BEACON_RATE_LIMIT_PER_MIN` (default 60) e `ONBOARDING_BEACON_RATE_LIMIT_BURST` (default 20)
- GET `/tokens/{token}/state` não escreve timestamps (invariante preservada)

## Decisões de Implementação

1. **Retry via `devkithttp.IdempotentNewRetryPolicy`**: cobre 5xx + 429 e não retenta 4xx. Isso reutiliza a política existente do devkit sem reimplementar backoff.

2. **tokenProvider com `sync.Mutex`**: garante thread-safety do cache sem goroutine extra. Token é revalidado comparando `time.Now().UTC()` com `expiresAt` calculado como `now + expiresIn - safetyMargin`.

3. **`golang.org/x/time/rate`**: era dependência transitiva; adicionada como direta via `go get`. Sem duplicação com o wrapper httpclient.

4. **FunnelToken fallback**: `tracking.s1` é o campo principal (confirmado pela techspec); se vazio, faz fallback para `tracking.src`. Isolado em `mapSale()` para troca cirúrgica (L-03).

5. **checkStatus separado do retry**: o retry é configurado na camada httpclient via `devkithttp.IdempotentNewRetryPolicy`. O `checkStatus` apenas classifica o erro final após o retry esgotar.

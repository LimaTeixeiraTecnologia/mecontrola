## Suposições
- `ProcessSubscriptionRenewed` quando a sub não existe no local cria placeholder com `FunnelToken{}` (zero-value); o campo será populado quando o `order_approved` correspondente chegar (idempotente por event_key)
- `ReconcileSubscriptions` recebe `db database.DBTX` nil; o checkpoint repository é acessado diretamente via factory com nil (padrão aceito para queries sem transação no projeto)
- Stale RENEWED quando sub está ACTIVE (target=ACTIVE) não é regressão por definição (targetStatus==currentStatus); apenas stale RENEWED quando sub está PAST_DUE ou CANCELED_PENDING (target=ACTIVE) é regressão

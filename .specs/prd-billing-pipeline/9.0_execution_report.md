## Decisões arquiteturais relevantes

- **depguard `platform-no-business-modules`:** `BillingSubsystem` colocado em `internal/billing/infrastructure/subsystem/` (não em `internal/platform/`). `cmd/server/server.go` faz o wiring via `runtime.NewApp` (lista explícita de subsistemas).
- **Lazy route registration:** `KiwifyRouteRegistrar` pre-criado com handler nil no `NewBillingSubsystem`; handler preenchido via `SetHandler` no `Start`. Chi recebe o registrar antes do Start via `NewServerSubsystemWithRegistrars`.
- **Outbox antes de Dispatcher.Start (RF-47):** `billingoutbox.RegisterHandlers` chamado dentro de `BillingSubsystem.Start`, que é iniciado antes do dispatcher via ordem de subsistemas em `runtime.NewApp`.
- **`identityUserResolverAdapter`:** adapta `identity.UserRepository.UpsertByWhatsAppNumber(ctx, number, now)` para `billing.UserResolver.UpsertByWhatsAppNumber(ctx, number)` injetando clock.
- **buildUseCases helper:** extraído de `Start` para manter function-length abaixo de 40 statements (revive linter).
- **KiwifyConfig.Safe():** retorna booleanos para secrets, nunca os valores em si — seguro para logs.
- **validateProductionKiwify skip:** quando `ClientID=="" && WebhookSecret=="" && ClientSecret==""` a integração Kiwify não está habilitada; validação de secrets obrigatórios é suprimida para backward compat.

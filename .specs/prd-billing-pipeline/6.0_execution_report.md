## Decisões Técnicas

1. **TxRunner[T]**: Interface genérica criada para abstrair `*database.UnitOfWork[T]` em testes unitários sem banco real. A implementação concreta (`*database.UnitOfWork[T]`) satisfaz a interface naturalmente em produção.

2. **applyCanonicalToSub refatorada**: Função original com complexidade ciclomática 19 (acima do limite de 15) foi decomposta em helpers `applySimple`, `applyWithPeriod`, `applyPurchaseApproved`, `applyRefund`.

3. **purchase_approved com sub existente**: Usa `sub.Renew()` em vez de `sub.Activate()`, pois `Renew` trata `Active→Active` como atualização de período sem validar transição de estado.

4. **buildDecision corrigido**: Lógica invertida — `!entitled` retorna denied primeiro, depois diferencia `granted` vs `grace` por status da subscription.

5. **buildExpiredSub via RehydrateSubscription**: `Active → Expired` não é transição legal; usa `RehydrateSubscription` para criar subscription em estado terminal diretamente.

6. **payload encoding no pacote usecases**: Estruturas `receivedEventPayload`, `encodeReceivedPayload`, `decodeReceivedPayload` definidas no pacote `usecases` para evitar dependência de infrastructure.

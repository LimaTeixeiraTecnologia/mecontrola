## Decisões Tomadas

- `EventHandlers` em `BillingModule` retorna `nil` (vazio): billing só produz eventos no MVP, não consome; conforme techspec §6.1.
- `SubscriptionEventProjector` resolve `user_id` via query direta em `billing_subscriptions` (cross-module read apenas para projeção; não é hot path); sem importação do módulo billing — apenas query SQL direta.
- `NotificationSender` stub no-op em `internal/billing/infrastructure/messaging/` (pacote neutro) em vez de no handler; permite substituição futura sem alterar interface.
- Cache LRU para `EntitlementReader` entregue como no-op TTL=0 (sem cache explícito); `EntitlementReader` delega direto ao Postgres. Trade-off: simplicidade > performance no MVP.

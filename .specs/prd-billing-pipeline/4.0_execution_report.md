- `Renew` para Active→Active (mesma assinatura renovando) trata o caso especialmente sem passar pelo StateMachine, pois a state machine não registra Active→Active como transição legal (não está em RF-17). Renew de PastDue→Active usa a state machine normalmente.

- `toIdentityStatus` como função privada no pacote `entities` (sem estado) aceita exceção pragmática R1 por ser helper puro dentro do mesmo arquivo que o agregado, seguindo o padrão já documentado em ADR-007 de identity.

## Riscos Residuais
- O `Renew` para Active→Active não passa pelo StateMachine. Se a state machine evoluir para permitir essa transição explicitamente, o método Renew deve ser refatorado para delegar ao StateMachine também neste caso.

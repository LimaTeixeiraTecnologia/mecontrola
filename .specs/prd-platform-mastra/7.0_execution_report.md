## Suposições
- O campo `event_id` mencionado na tarefa para idempotência não existe na tabela `platform_scorer_results` (schema DDL não contém unique constraint); a inserção é idempotente por `id` (uuid gerado pelo runner).
- `ScorerRunner.Observe` é não-bloqueante (fire-and-forget via canal com buffer); falha silenciosa quando canal está cheio (drop) — comportamento aceitável para evals fora do caminho crítico.
- Integration test requer `//go:build integration` e testcontainers; não roda no CI padrão.

## Riscos Residuais
- Ratio sampling usa `math/rand` sem seed fixo; em testes de ratio o resultado é não-determinístico. Mitigação: testes cobrem `always` e `never` (determinísticos); ratio não é testado deterministicamente.
- O runner usa `context.Background()` interno no `processJob`, desacoplando do contexto original (by design, pois Observe é fire-and-forget).

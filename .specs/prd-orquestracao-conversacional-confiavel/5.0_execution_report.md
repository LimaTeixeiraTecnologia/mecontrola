## Riscos Residuais
- `no_internal_terms` usa lista conservadora de termos (incluindo nomes literais de tools); pode gerar falso positivo raro se uma resposta legítima mencionar um termo da lista em contexto não técnico — mitigado por ser sinal de medição contínua (ADR-004), não enforcement, e por "Revisão Futura" do ADR-004 prever ajuste do conjunto de marcadores conforme o golden evoluir (fora do escopo desta tarefa).
- `verbatim_required` depende de `sample.Metadata["verbatim_text"]`, uma convenção nova (não havia precedente no código para esse conceito — grep confirmou ausência de "verbatim" como chave de metadata antes desta tarefa). Hoje nada popula essa chave em produção (o campo `verbatimText` das tools flui para `agent.Result.Content` via `extractVerbatim`, não para `RunSample.Metadata`), então o scorer sempre retorna `1.0` trivial em prod até um consumidor popular a metadata — comportamento documentado no ADR-004 ("compara com verbatimText da tool") e coerente com o escopo desta tarefa (não popular pipelines de metadata é responsabilidade de tarefas futuras/harness golden, RF-30 pede o scorer, não o wiring de metadata).
- A mudança de assinatura de `Hooks.AfterTool` é uma extensão de contrato público do substrato `internal/platform/agent`; embora estritamente aditiva e com todos os call-sites/mocks atualizados nesta tarefa, qualquer consumidor externo futuro de `Hooks` (fora deste repositório) precisaria de atualização — não é o caso aqui (todos os implementadores estão neste mesmo módulo).

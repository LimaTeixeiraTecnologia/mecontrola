## Resumo

Removidos os feature flags `kernelEnabled`/`EnableKernel`/`TransactionsWriteEnabled`, o fallback de orçamento legado (`BudgetConfigurator`) e a path legacy de despacho de escrita. O kernel de workflow é agora o único caminho para operações de escrita de transações (`KindRecordExpense`, `KindRecordIncome`, `KindRecordCardPurchase`). Os testes de paridade foram migrados para `transactions_write_kernel_test.go` e `parity_test.go` foi deletado.

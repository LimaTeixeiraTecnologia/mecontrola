## Suposições
- O bucket S3 `mecontrola-backups-660838763799-use1` e o IAM user `mecontrola-staging-s3` são os recursos reais de staging criados via Terraform.
- `PGBACKREST_REPO1_CIPHER_PASS` não estava definido no `.env` real; uma senha temporária de 64 caracteres foi gerada (`openssl rand -base64 48`) apenas para os testes de re-validação. O `.env` de produção/staging deve ser preenchido com uma senha forte persistente.
- O script `pgbackrest-setup.sh` depende das variáveis `PGBACKREST_S3_*` (sem `REPO1`) como entrada e exporta `PGBACKREST_REPO1_S3_*` para o pgBackRest.

## Riscos Residuais
- `PGBACKREST_REPO1_CIPHER_PASS` está vazio no `.env` real; sem preenchê-lo, o pgBackRest falha com `environment variable 'repo1-cipher-pass' must have a value`.
- pgBackRest 2.58 emite warning de depreciação sobre `stop-auto=y`; não é blocker, mas deve ser removido em futura revisão.
- O container de restore requer todas as credenciais S3 (`PGBACKREST_REPO1_S3_*`) e o cipher pass no ambiente para que o `archive-get` do PostgreSQL recovery funcione; isso deve estar documentado no runbook de restore PITR.

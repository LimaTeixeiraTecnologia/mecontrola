## Riscos Residuais
- Docker Swarm single-node não oferece HA real; falha do host ainda indisponibiliza a aplicação (mitigado por backups S3).
- Oversubscription de CPU na Hostinger KVM2 com 2+2 réplicas pode causar OOM/latência; monitorar thresholds e planejar upgrade para KVM4/KVM8.
- Migração de Compose para Swarm não tem snapshot/rollback formal; requer janela de manutenção e backups validados.
- `depends_on` no Swarm não suporta `condition: service_healthy`; a ordem de startup depende de healthchecks + restart policy.

## Decisoes Tecnicas

1. **Parent-based sampling (7.1):** devkit-go v0.5.5 mapeia `TraceIDRatioBased` para taxas 0..1; nao expoe `ParentBased`. Resolucao: `OTEL_TRACE_SAMPLE_RATE=1` â†’ `AlwaysSample()` nos 4 servicos. O comportamento parent-based emerge naturalmente via propagacao do traceparent (spans do worker sao filhos dos spans do server).

2. **Traceparent no hop async (7.2):** `otel.GetTextMapPropagator()` usa o propagador global ja registrado pelo `NewProvider` do devkit-go com `RegisterGlobal: true`. Nao e necessario configurar propagador adicional em producao. Em testes, `gotel.SetTextMapPropagator(...)` com `defer` para isolamento.

3. **lag em `mark()` (7.6):** `row.OccurredAt` ja esta disponivel no `DispatcherJob.mark()` apos `MarkPublished`. Guard `!row.OccurredAt.IsZero()` evita NaN quando campo ausente.

4. **Labels de cardinalidade controlada:** `outbox_lag_seconds` usa `event_type`; `agents_whatsapp_inbound_timeout_total` usa `channel`; `workflow_resumed_on_conflict_total` usa `workflow`. Nenhum `user_id`, `correlation_key` ou `category_id` como label de metrica.

## Suposições

- `WhatsAppGateway` e `OutboxPublisher` são fornecidos pelo `cmd/server` para o canal WhatsApp estar ativo. Se nil, `WhatsAppAgentRoute` não é registrado (comportamento defensivo).
- `ResourceID` = `principal.UserID.String()` e `ThreadID` = `peer` (número E164) são suficientes para chaves opacas de Thread/Run na plataforma, conforme ADR-003.
- O gateway WhatsApp existente (`SendTextMessage`) é thread-safe e não requer wiring adicional nesta tarefa.

## Riscos Residuais

- Idempotência do consumer no cenário de retry de outbox (falha pós-dispatch, pré-ack): o runtime criará um segundo Run para o mesmo `MessageID`. Mitigação futura: dedup por `message_id` em `RunStore` (fora do escopo MVP).
- `WhatsAppAgentRoute` retorna `OutcomeInvalid` se `OutboxPublisher` ou `WhatsAppGateway` são nil.

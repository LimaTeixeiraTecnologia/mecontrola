## Riscos Residuais

1. **Testes de integração E2E requerem banco de dados real**: Os testes usam testcontainers-go e são marcados com build tag `//go:build integration`, portanto não rodam no `go test ./...` padrão.

2. **golangci-lint reporta issues não-bloqueantes**: As issues identificadas (errcheck, goimports, revive function-length, unused) são preexistentes ou relacionadas a mocks gerados automaticamente.

3. **CC-V3 e CC-V4 parcialmente cobertos**: A validação completa de migration editorial requer setup de migration em ambiente de teste, o que está fora do escopo desta tarefa.

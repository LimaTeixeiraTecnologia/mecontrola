## Decisões Técnicas

1. **IDs Determinísticos:** Os UUIDs dos termos canônicos são os mesmos das categorias (UUIDv5 do slug), garantindo idempotência e previsibilidade.

2. **Seed Mínimo (MVP):** A tarefa 3.0 implementa apenas os canônicos obrigatórios (RF-32). Aliases, frases (RF-33) e termos ambíguos (RF-34) são opcionais para o MVP e podem ser adicionados em migrations futuras.

3. **Normalização:** A coluna `term_normalized` é gerada automaticamente via `lower(unaccent(term))`, garantindo busca case-insensitive e accent-insensitive.

4. **Versionamento:** A migration incrementa a `category_editorial_version`, permitindo cache e invalidação via ETag.

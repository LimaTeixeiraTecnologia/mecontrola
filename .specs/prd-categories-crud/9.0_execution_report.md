## RF-42: Proteção Contra Log de Termo Bruto

**Implementação:** O termo de busca (`q`) nunca é logado ou usado como label de métrica.

- Logs contêm apenas: `endpoint`, `method`, `outcome`, `duration_ms`
- Métricas usam `q_len_bucket` (tamanho da query normalizada), nunca o termo bruto
- O teste `TestLogs_DoNotContainRawQuery` valida essa proteção

## Notas

- As métricas são registradas usando a API `Add`/`Increment` e `Record` do devkit-go com contexto
- O `q_len_bucket` é calculado a partir do `len(q)` após trim e normalização (remoção de caracteres não alfanuméricos)
- O `signal_type_top` é preenchido apenas quando há um candidato vencedor (não ambíguo)
- Todos os handlers registram métricas de contador e histograma para todos os outcomes

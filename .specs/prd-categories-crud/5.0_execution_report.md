## Problemas Encontrados

### 1. Migration de Seed Inconsistente
A migration `000006_seed_dictionary.up.sql` foi alterada durante o desenvolvimento e agora contém `category_id`s que não existem na tabela `categories` (ex: `4e7e9e7d-8c8d-5e1c-9f2b-5f8f3f9f3f3f`). Isso causa erro de foreign key constraint quando a migration é executada.

**Impacto:** Os testes de integração que dependem do seed completo falham na etapa de setup do banco.

**Solução recomendada:** Corrigir a migration `000006_seed_dictionary.up.sql` para usar os IDs corretos das categorias existentes no seed `000005_seed_categories.up.sql`.

### 2. Collation pt_BR Não Disponível
A techspec especifica `COLLATE "pt_BR"` para ordenação, mas a imagem `postgres:16-alpine` usada nos testes não possui esse locale.

**Solução aplicada:** Removido o COLLATE das queries para permitir execução dos testes. Em produção, o PostgreSQL deve ter o locale pt_BR configurado.

## Implementação

### OnboardingCompletedConsumer

- Decodifica `outbox.Envelope` → `onboardingCompletedPayload{UserID}`
- Chama `onboardingContextReader.Execute` (interface thin sobre `GetOnboardingContext`)
- Se `!snapshot.Found` → retorna nil (no-op)
- Se WM já existe e tem conteúdo → retorna nil (idempotência)
- Constrói WM via `buildOnboardingWM` e chama `wmRepo.Upsert`
- `decodeFails` counter para erros de decode/parse UUID
- Span de observabilidade: `agent.consumer.onboarding_completed.handle`

### Remoção de WM inline

- `synthesizeAndStoreWM` e `buildWMFromSnapshot` removidas de `onboarding_tool_dispatcher.go`
- `contextReader` e `wmWriter` removidos do struct e constructor do dispatcher
- `attachOnboardingLLM` em `module.go` atualizado para não passar esses campos

### Wiring

- `buildEventHandlers` registra `{EventType: "onboarding.completed", Handler: NewOnboardingCompletedConsumer(...)}` quando `b.onboardingLLM != nil && b.onboardingLLM.GetContext != nil && b.wmRepo != nil`

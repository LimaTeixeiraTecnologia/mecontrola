## Decisões de Implementação

1. `WhatsAppWebhookRouter` aceita `func(http.Handler) http.Handler` em vez de `*middleware.RateLimiter` para evitar cross-module boundary violation (`identity` não pode importar `onboarding`).
2. O `captureWriter` intercepta o status HTTP para detectar 429 e chamar o callback de métrica — sem alterar o comportamento existente do `RateLimiter`.
3. O callback recebe `context.Background()` pois o contexto da requisição já expirou no momento da chamada do counter (a resposta 429 foi escrita).

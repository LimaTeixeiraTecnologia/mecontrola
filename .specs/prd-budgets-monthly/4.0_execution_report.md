## Riscos Residuais

- O `ThresholdStateRepository.UpsertIfTransition` usa uma CTE complexa para detectar transição atomicamente. Funciona corretamente em todos os cenários testados; em caso de alta concorrência simultânea o resultado da coluna `before_crossed` pode ser NULL (nova linha) — tratado como transição (retorna `true`), comportamento correto.
- O cursor de `AlertRepository.ListForUser` usa posição por `(created_at, id)` com UUID aleatório — pode haver ties no mesmo `created_at`. Resolvido com ordenação secundária por `id DESC`.

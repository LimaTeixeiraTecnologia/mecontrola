# BDD: Módulo Billing
**Data:** 2026-06-17
**Status:** MVP Robust / Production-Ready
**Referência:** Domain Modeling Made Functional (DMMF)

## Objetivo
Garantir a integridade do ciclo de vida das assinaturas processadas via webhook (ex: Kiwify), validando estados de domínio, persistência e eventos emitidos.

## Fluxo 1: Processamento de Venda Aprovada (Kiwify)
**Funcionalidade:** Ativação de nova assinatura após aprovação de pagamento.

**Cenário:** Sucesso na ativação de assinatura via Webhook
- **Dado** que um webhook de 'sale_approved' é recebido da Kiwify
- **E** o token de funil é válido
- **Quando** o sistema processa o webhook
- **Então** uma nova `Subscription` deve ser criada no banco de dados com status `Active`
- **E** um evento de domínio `SubscriptionActivated` deve ser emitido
- **E** os detalhes do pagamento devem ser registrados para reconciliação.

## Fluxo 2: Cancelamento de Assinatura
**Funcionalidade:** Gestão de encerramento de contrato.

**Cenário:** Assinatura cancelada pelo cliente
- **Dado** que uma assinatura está no estado `Active`
- **Quando** um evento de cancelamento é processado
- **Então** o status da `Subscription` deve transicionar para `Canceled`
- **E** a data de encerramento deve ser registrada
- **E** o acesso do usuário deve ser revogado após o período de carência, se aplicável.

## Fluxo 3: Renovação de Assinatura
**Funcionalidade:** Extensão do período de vigência.

**Cenário:** Renovação automática confirmada
- **Dado** que uma assinatura está próxima do vencimento
- **Quando** um webhook de 'subscription_renewed' é recebido
- **Então** a validade da `Subscription` deve ser estendida
- **E** o histórico de renovações deve ser atualizado no banco de dados.

## Regras de Domínio (DMMF)
- **Estados Ilegais:** Uma assinatura não pode estar `Active` sem uma data de início válida.
- **Smart Constructors:** O ID da assinatura Kiwify deve ser validado no momento da reidratação/criação.
- **Transições:** Apenas assinaturas `Late` ou `Active` podem transicionar para `Canceled`.

## Validação de Produção
- [ ] Verificar integridade da chave de idempotência do webhook.
- [ ] Validar persistência no esquema `billing.subscriptions`.
- [ ] Garantir que eventos de domínio sejam capturados pelo barramento de eventos.
